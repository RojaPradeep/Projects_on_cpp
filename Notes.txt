   malloc:int *arr=(int*) malloc(sizeof(int));     1 paramter ,no intialization ->garbage value ,if fails to alocate it return  null ,faster
   calloc: arr=(int*) calloc(5,sizeof(int));    2 paratermer, initialization to 0,null,slow ,block of memory is allocated with 0 values
   realloc:arr =(int*)realloc (ptr,sizeof(int)); 2 paraterms[old ptr,size], used to resize the existing memory block,no initialization only copying the existing old data ,slower

SIP :
 What is SIP?
    SIP is Sesssion Initiative Protocol which is used to initiate, manage and terminate the communication session over a IP networks.
 What is RTP?
    RTP is Real Transport Protocol actual transfer of data audio/vedio interms of packets once the connection session is established during a call.


Difference b/w SIP and RTP
             SIP                                                                       RTP 
    Set up the call establish                               Transfer the data once the connection is established

SBC :
  What is SBC?
     Session Border Controller : 
	     It is a network device which allows the secure control and manage SIP based audio/vedio voip based connections between different networks.
		 
		 Advantages:
		 1.Security : avoid sip spoofing, hiding internal ip
		 2.Call routing: decides the best route to comminicate
		 3. quality of service: provide good audio ,vedio quality over call
		 4.Nat traversal: traversal of call over firwall
		 5.Interoperability:convert protocol/codees betweeen different system

   Session Description Protocol:
   It is format used to decscribe the multimedia [audio/vedio] session.

FieldMeaning
v=Version
o=Owner /s session ID
s=Session name
c=Connection (IP address)
m=Media type + port
a=Attributes (codec info)

Flow of Call?
      Fisrt SIP starts the session by using INVITE message 
	  SDP has codec information, media type info ,port number, session name 
	  Receiver will responds to the SDP 
	  SIP connection establish
	  Through RTP data / Media transfer via UDP Protocol.
